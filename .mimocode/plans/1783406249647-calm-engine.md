# رفع باگ: telegramId بزرگتر از INT4

## ریشه باگ

آیدی تلگرام کاربر `6364442558` بزرگتر از حداکثر مقدار `Int` PostgreSQL (2,147,483,647) هست. فیلد `telegramId` در Prisma schema به صورت `Int` تعریف شده → upsert با خطا مواجه میشه → احراز هویت fail می‌کنه → کاربر اسپینر ابدی می‌بینه.

## تغییرات

### ۱. `server/prisma/schema.prisma` — تغییر telegramId به BigInt

```prisma
telegramId  BigInt    @unique    // ← از Int به BigInt
```

### ۲. ایجاد migration

```bash
npx prisma migrate dev --name change-telegramId-to-bigint
```

### ۳. `server/src/services/auth.service.ts` — تبدیل BigInt به number

Prisma BigInt رو به صورت BigInt object برمی‌گردونه، نه number. باید تبدیل کنیم چون Telegram ID همیشه در محدوده safe integer جا می‌شه.

- خط ۷۳: `const { id, ... } = data.user` → بعدش `Number(id)` رو استفاده کن
- خط ۷۷: `where: { telegramId: Number(id) }`
- خط ۸۵: `telegramId: Number(id)`
- خط ۹۶: `telegramId: Number(user.telegramId)` (Prisma BigInt → number)

### ۴. `server/src/middleware/auth.ts` — تبدیل BigInt در JWT

- خط ۸: `telegramId: number` (باقی بمونه — JWT number هست)
- خط ۴۵: `telegramId: Number(payload.telegramId)` یا `Number(user.telegramId)`

### ۵. `server/src/routes/queue.ts` — تبدیل BigInt در مقایسه

- خط ۲۰: `e.user.telegramId === req.user!.telegramId` — چون هر دو number هستن، کار می‌کنه. ولی باید مطمئن بشیم `req.user.telegramId` از ماینتل مiddleware به صورت number میاد.

### ۶. `server/src/seed.ts` — بروزرسانی telegramId ها

تبدیل telegramId ها به BigInt-compatible: `BigInt(100000001)` یا `100000001n`

### ۷. `server/src/services/admin.service.ts` — خط ۲۹

`log.user?.telegramId?.toString()` — Prisma BigInt خودش `.toString()` داره، مشکلی نیست.

## فایل‌های تغییر یافته

| فایل | تغییر |
|------|-------|
| `server/prisma/schema.prisma` | `telegramId Int` → `BigInt` |
| `server/src/services/auth.service.ts` | `Number()` برای تبدیل BigInt |
| `server/src/middleware/auth.ts` | تبدیل BigInt در req.user |
| `server/src/seed.ts` | بروزرسانی telegramId ها |

## بررسی

1. `npx prisma migrate dev` — migration اجرا بشه
2. `npx prisma db seed` — seed مجدداً اجرا بشه (اختیاری)
3. سرور رو restart کن و با یوزر دیگه تست کن
4. چک کن که احراز هویت موفق باشه و داشبورد نشون داده بشه
